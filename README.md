# Excel Formula Tutorial Project

## 📂 Project Overview
This project demonstrates the use of **common Excel formulas** applied to a dataset of fictional employees (inspired by *The Office* characters).  
The dataset includes:
- Employee details (ID, Name, Age, Gender, Job Title, Salary)  
- Employment dates (StartDate, EndDate)  
- Additional fields for formula demonstrations  

The goal is to showcase **Excel formula applications** for data cleaning, transformation, and analysis.

---

## 🧮 Formulas Covered
- **Max-Min**: Identify maximum and minimum values in salary or age.  
- **IF / IFS**: Apply conditional logic to categorize employees.  
- **LEN**: Measure string length (useful for validating IDs or SSNs).  
- **LEFT / RIGHT**: Extract substrings (e.g., email domains, initials).  
- **TEXT (DateToText)**: Convert dates into custom text formats.  
- **TRIM**: Remove unwanted spaces from text fields.  
- **SUBSTITUTE**: Replace specific text instances within strings.  
- **SUM / SUMIF / SUMIFS**: Aggregate salary data with conditions.  
- **COUNT / COUNTIF / COUNTIFS**: Count records based on criteria.  
- **CONCATENATE**: Combine first and last names into full names.  
- **DAYS / NETWORKDAYS**: Calculate duration of employment (calendar vs. working days).  

---

## 📊 Learning Outcomes
This project highlights:
- Practical use of **Excel formulas** for everyday business tasks.  
- Data validation and cleaning techniques (TRIM, LEN, SUBSTITUTE).  
- Analytical functions for summarizing and comparing data (SUMIFS, COUNTIFS).  
- Date and text manipulation for reporting (TEXT, DAYS, NETWORKDAYS).

# 🚴 Bike Buyers Analysis Dashboard

## 📌 Project Overview

This project analyzes customer demographic and lifestyle data to identify factors that influence bicycle purchases. The dataset was cleaned, transformed, and analyzed using Microsoft Excel to uncover patterns and trends among customers who purchased bikes and those who did not.

The project demonstrates practical Excel skills including data cleaning, transformation, analysis, visualization, and dashboard creation.

---

## 🎯 Project Objectives

The main objectives of this project were to:

- Clean and prepare raw customer data for analysis.
- Identify factors influencing bike purchases.
- Create Pivot Tables to summarize key insights.
- Develop visualizations to communicate findings.
- Build a dashboard to present results in an easy-to-understand format.

---

## 📂 Dataset Description

The dataset contains customer information such as:

| Column | Description |
|----------|------------|
| ID | Unique customer identifier |
| Marital Status | Customer marital status |
| Gender | Customer gender |
| Income | Annual income |
| Children | Number of children |
| Education | Education level |
| Occupation | Customer occupation |
| Home Owner | Home ownership status |
| Cars | Number of cars owned |
| Commute Distance | Distance traveled to work |
| Region | Customer region |
| Age | Customer age |
| Purchased Bike | Indicates whether a customer purchased a bike |

---

## 🧹 Data Cleaning Process

Before analysis, the dataset was cleaned to improve accuracy and consistency.

### Cleaning Tasks Performed

#### 1. Data Inspection
- Reviewed the dataset for inconsistencies and errors.
- Examined data structure and column formats.

#### 2. Duplicate Verification
- Checked for duplicate records.
- Ensured each customer entry remained unique.

#### 3. Standardization
- Standardized categorical values for consistency.
- Verified formatting across multiple columns.

#### 4. Data Validation
- Confirmed numerical fields contained valid values.
- Checked for missing or incomplete records.

#### 5. Feature Engineering
Additional fields were created to support analysis.

Examples include:
- Age Brackets
- Customer Segmentation Categories

These additions helped improve the effectiveness of Pivot Table analysis.

---

## 📊 Data Analysis

Pivot Tables were created to answer key business questions and identify trends in bike purchasing behavior.

### Income Analysis
- Examined the relationship between customer income and bike purchases.
- Identified purchasing patterns across income groups.

### Age Analysis
- Analyzed which age groups were most likely to purchase bikes.
- Grouped customers into age brackets for easier comparison.

### Occupation Analysis
- Compared bike purchases across different occupations.
- Identified occupations with higher purchase rates.

### Commute Distance Analysis
- Investigated whether commute distance influenced purchasing decisions.

### Regional Analysis
- Compared bike purchases across geographical regions.

### Marital Status Analysis
- Evaluated differences in purchasing behavior based on marital status.

---

## ⚠️ Dashboard Limitation

The dashboard included in this project is **not fully interactive**.

This limitation exists because the project was developed using the **free web version of Microsoft Excel (Excel Online)** rather than the **desktop version of Microsoft Excel**.

The free web version has several restrictions, including:

- No Pivot Chart Analyze tab
- Limited Pivot Chart functionality
- Reduced dashboard customization options
- Limited support for interactive dashboard features such as slicers

As a result:

- The dashboard successfully displays analytical insights and visualizations.
- Pivot Tables and charts are functional.
- Advanced interactive dashboard features could not be implemented.

If the project is opened and further developed using the desktop version of Microsoft Excel, additional dashboard functionality and interactivity can be added.
